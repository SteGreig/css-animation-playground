Try some of these animate.css effects below and see how the keyframes translate to the resulting animation (don't forget to play with the duration and other animation controls).

animate.css is a CSS library packed with tons of little keyframe based animations. You can simply include the file in your page, and add the appropriate classes (such as .fadeIn to the element(s) you want to animate.

However, most of the time you will only want to use 1 or 2 of the animations from the library, meaning it'd make more sense to include the keyframes from these animations in your main stylesheet, subsequently saving an extra http request.

animation-name (required)

This is the name that you gave to your set of keyframes and is required for your animation to work

@keyframes myAnim { }

animation-duration (required)

This defines how long the animation should last and is required for your animation to work

It accepts values in miliseconds or seconds, e.g.

300ms

0.5s

3s

animation-iteration-count (optional)

This defines how many times you want the animation to run

It accepts number values and the infinite keyword

The default value is 1

You can also use decimal values such as 1.5, which would run the animation 1 and a half times

animation-delay (optional)

Use this property to delay the start of your animation

It accepts values in miliseconds or seconds, e.g.

300ms

0.5s

3s

animation-direction (optional)

This property states whether the animation should play normally or in reverse

The possible values are:

normal (default value)

reverse Plays the animation backwards

alternate If the animation is set to run more than once, this value would play it forwards, then backwards and so on

alternate-reverse If the animation is set to run more than once, this value would play it backwards, then forwards and so on

animation-play-state (optional)

This property allows you to pause your animation and is most useful on hover and on other trigger events

The possible values are:

running (default value)

paused Pauses the animation

animation-fill-mode (optional)

By default, if your animation doesn't finish in the same place as it starts, it snaps back to its original styling on completion.

This property allows it to keep the styles set in your last keyframe.

The possible values are:

none (default value)

forwards ensures that your element maintains the styles specified in the last keyframe after the animation finishes

backwards applies if you have specified an animation-delay value because it ensures the styles from the first keyframe are applied immediately, even before the animation has begun

both applies the rules from the previous two values

animation-timing-function (optional)

This property allows you to define where the animation accelerates and decelerates over the specified duration, and can be very powerful.

There are 5 keyword values which are all essentially shorthands for a cubic-bezier() function.

You can use this cubic-bezier() function to create custom timing functions (Create Your Own)

Bear in mind that the timing function is applied to each keyframe rather than the animation as a whole, so the ease-in value for example would ease into each keyframe

The possible values are:

ease (default value)

ease-in

ease-out

ease-in-out

linear

cubic-bezier(0,0,0,0)

steps(1) The steps() function causes your animation to play over a specified number of steps, jumping from one keyframe to the next, rather than smoothly transitioning between each keyframe

I explain animation timing functions in much greater depth in this article

animation (shorthand)

The values for all of the previous properties can be combined into a single animation property

The only required values for this property are animation-name and animation-duration

The syntax for this property is very lenient in terms of what order the values can be stated - the only rule is that if you include both an animation-duration and animation-delay value, the duration value must come before the delay value
